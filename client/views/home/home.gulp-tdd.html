gulp tdd

Run all server tests once and then watch for code changes.

If test code (*.spec.js) is changed, then the tests in that file are run.

If application code is changed then the tests in the associated .spec.js file are run (if one can be found). It must be named like the application code file only with .spec just before the .js.

NB: It doesn't support running client tests one file at a time: the --client option runs all the tests on each change.

  1. gulp tdd --client (run client tests and watch for changes, re-running all the tests on code change)
  2. gulp tdd --server (this is the default behaviour as described above)
